MinuteMind — Screen Specifications

MAUI translation of each Stitch mockup with component breakdown

1. Dashboard (Home Tab)

Layout
  • ScrollView → VerticalStackLayout
  • Header: Grid (hamburger + "MinuteMind" label + avatar Border)
  • Hero: Label "Capture Clarity." (PlusJakartaBold, 34sp) + subtitle (Inter, 15sp)
  • Action Cards: 2-column Grid, equal aspect ratio
  • Record card: Border with gradient bg (#005FAA→#0078D4), rounded 32dp
  • Upload card: Border white bg, ghost shadow, rounded 32dp
  • Recent Meetings: CollectionView with DataTemplate
  • Each item: Border (white, rounded 24dp) → Grid (icon container + title/date/badge + chevron)
  • AI Pulse footer: decorative Label
ViewModel Bindings
  • RecentMeetings — ObservableCollection<Meeting> (last 5)
  • RecordCommand → navigate to RecordingPage
  • UploadCommand → FilePicker then ProcessingPage
  • ViewMeetingCommand → MinutesPage with meeting ID
  • ViewAllCommand → switch to Search tab
Status Badges
  • "MINUTES READY" → SecondaryContainer bg, pill shape
  • "TRANSCRIBED" → SurfaceContainerHigh bg, muted
  • "PROCESSING" → Primary bg with pulse animation

2. Recording Page

Layout
  • Full-screen, tab bar hidden (Shell.TabBarIsVisible=false)
  • Header: same app bar
  • "AI TRANSCRIPTION ACTIVE" chip: Border with SecondaryContainer/30 bg
  • Meeting title: Label (PlusJakarta, 28sp bold)
  • Metadata: "Started at 10:30 AM • 4 Participants Detected"
  • Waveform: Custom GraphicsView (WaveformView) — 13 animated bars
  • Timer: Label (PlusJakarta, 64sp ExtraBold, tabular figures)
  • Stop button: 96dp circle Border with pulsing red animation
  • Pause + Stop Recording: HorizontalStackLayout of Buttons
  • Toast: "Capturing from Internal Mic" — fixed bottom Border
  • Live Insight: Border card with real-time transcript preview
ViewModel Bindings
  • IsRecording, IsPaused — state booleans
  • ElapsedTime — TimeSpan, formatted HH:mm:ss
  • WaveformLevels — float[] for bar heights
  • LiveInsightText — latest transcript snippet
  • MeetingTitle — editable string
  • PauseCommand, StopCommand
  • Uses IAudioRecorderService for actual recording
Animations
  • Waveform bars: random height changes via timer (CommunityToolkit animations)
  • Stop button: pulsing red ring (ScaleTo animation loop)
  • Red dot on Live Insight: opacity pulse

3. Processing Page

Layout
  • Full-screen, tab bar hidden, back nav disabled
  • "Processing Meeting..." headline (PlusJakarta, 28sp bold)
  • Central spinner: 192dp Border circle with rotating arc (custom GraphicsView or animated Border)
  • Inner: mic icon + "ANALYZING" label
  • 3 step cards in VerticalStackLayout:
  • Step 1 (done): white card, checkmark circle, "Uploading audio... 2.4MB complete"
  • Step 2 (active): white card with blue left border (4dp), spinner icon, bold blue text
  • Step 3 (pending): muted card at 60% opacity
  • Footer: "POWERED BY MINUTEMIND INTELLIGENCE V4.2"
ViewModel Bindings
  • CurrentStep — enum (Uploading, Transcribing, Generating)
  • Steps — ObservableCollection with status per step
  • Auto-advances through steps with mock delays (2s, 3s, 2s)
  • On completion → auto-navigates to TranscriptPage
  • Uses ITranscriptionService + IMinutesGeneratorService (mocked)

4. Transcript Page

Layout
  • Header: back arrow + "Transcript" + search + avatar
  • Meeting header: "LIVE SESSION" label, title (36sp), speaker avatars overlapping, metadata
  • CollectionView of transcript segments:
  • Each segment: Border (white, rounded 12dp, ghost shadow)
  • → timestamp badge, speaker indicator (colored square with "S1"/"S2"), speaker name + text
  • AI Insight breaks: centered chip between segments ("AI Identified Key Topic: Technical Debt")
  • Floating "Generate Minutes" FAB: gradient pill button, fixed bottom center
ViewModel Bindings
  • Meeting — Meeting model with title, date, duration
  • Segments — ObservableCollection of transcript items (includes AI insight items as a different type)
  • GenerateMinutesCommand → ProcessingPage then MinutesPage
  • DataTemplateSelector to differentiate speaker segments from AI insight breaks

5. Minutes Page

Layout (Bento Grid)
  • Header: hamburger + "Meeting Minutes" + avatar
  • Hero section: blue left border (4dp), meeting type label, title (36sp), date+duration, Edit + Export buttons
  • Attendees card: white bg, icon, list of names with colored initial circles
  • Key Discussion Points card: white bg, bullet list with primary-colored dots
  • Decisions card: FULL PRIMARY BG (#005FAA), white text, 2-col grid of frosted glass sub-cards with category labels
  • Action Items: SurfaceContainerLow bg, checklist with checkboxes, assignee + due date
  • "AI-Generated Intelligence Layer" badge at bottom
  • Bottom tab bar visible
ViewModel Bindings
  • Minutes — MeetingMinutes model
  • Attendees — ObservableCollection
  • DiscussionPoints — ObservableCollection<string>
  • Decisions — ObservableCollection (Category + Text)
  • ActionItems — ObservableCollection<ActionItem>
  • ToggleActionItemCommand — marks item complete in SQLite
  • EditCommand → EditMinutesPage
  • ExportCommand → ExportPage

6. Edit Minutes Page

Layout
  • Header: X close + "Edit Minutes" + "Save Changes" button (blue pill)
  • Title: Entry control styled as transparent borderless input (30sp ExtraBold)
  • "AI-REFINED" badge + date label
  • Rich text area: WebView with contenteditable HTML
  • Pre-loaded with meeting minutes formatted as HTML
  • Fixed bottom toolbar: B / I / U / bullet / numbered / quote / link buttons + "AI Polish" pill
  • Toolbar sends JS commands to WebView (execCommand)
ViewModel Bindings
  • Title — two-way bound Entry
  • HtmlContent — string loaded into WebView
  • SaveCommand → extracts HTML from WebView, saves to SQLite, pops back
  • FormatBoldCommand, FormatItalicCommand, etc. → invoke JS
  • AiPolishCommand → mock service (future: real AI rewrite)
  • HasChanges — tracks dirty state for save button

7. Export Page

Layout
  • Header: X close + "Export Minutes" + avatar
  • Success state: checkmark circle (SecondaryContainer bg) + "Your minutes are ready to share." (28sp bold) + subtitle
  • Document preview card: Border (white, rounded 32dp)
  • → "DOCUMENT PREVIEW" label, filename, date+size, PDF icon
  • → Visual placeholder: skeleton bars mimicking document content
  • "Download PDF" button: full-width gradient pill, 64dp height
  • "SHARE WITH TEAM" section: 4 icon buttons (Slack/Email/Drive/More) as rounded squares
  • Bottom tab bar visible
ViewModel Bindings
  • DocumentName, ExportDate, FileSize
  • DownloadPdfCommand → IPdfExportService generates PDF, saves to device
  • ShareCommand → MAUI Share API (native share sheet)
  • Share buttons: Slack/Email/Drive use Share.RequestAsync with specific targets where available, fallback to generic share

8. Meetings Page (Search Tab) & 9. Settings Page

Meetings Page
  • SearchBar at top with mockup styling
  • Filter chips: All / Transcribed / Minutes Ready
  • CollectionView reusing same meeting card template as Dashboard
  • Empty state: single large "Quiet." label per design system
  • Swipe-to-delete on meeting items
Settings Page
  • Grouped sections: Audio (quality picker), Storage (clear cache), About
  • Future-ready: AI Provider config, Theme toggle, Account
  • Uses standard MAUI TableView/ListView styling
  • Minimal — not a priority screen for this version
